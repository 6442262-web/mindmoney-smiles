# การทดลองที่ 3–5: ทดสอบระบบอัตโนมัติ (รันเองได้ ไม่ต้องใช้คน/AI จริง)

การทดลองกลุ่มนี้ **AI ผู้ช่วยรันให้แล้วเมื่อ 10 ก.ค. 2026** — ผลอยู่ใน `RESULTS.md` ข้าง ๆ
ทุกอย่างทำซ้ำได้บนเครื่องใดก็ได้ที่ clone repo + `npm install` แล้ว (ต้องมี Chromium ของ Playwright)

## มีอะไรบ้าง

| การทดลอง | วัดอะไร | คำสั่ง |
|---|---|---|
| 3. ความน่าเชื่อถือของโปรแกรม | % เคสทดสอบอัตโนมัติ (unit test) ที่ผ่าน | `npx vitest run` |
| 4. การทำงานครบวงจร (E2E) | ฟีเจอร์หลักใช้ได้จริงบนเบราว์เซอร์ | ดูขั้นตอนล่าง |
| 5. ความเร็วโหลดแอป | FCP / DOM พร้อม / โหลดครบ (ms) | ดูขั้นตอนล่าง |
| 7. ความเร็วบนเน็ตจำลอง 3G/4G | FCP ต่อโปรไฟล์เครือข่าย | `node experiments/e2e/net-perf.mjs` |
| 8. ไฟล์ Excel ใหญ่ 100–10,000 แถว | เวลา parse + ความถูกต้อง | `node experiments/e2e/xlsx-scale.mjs` |

## วิธีรัน E2E (การทดลอง 4–5)

ต้องเปิด 3 เทอร์มินัล (หรือรัน background):

```sh
# เทอร์มินัล 1 — ฐานข้อมูลจำลอง (in-memory ไม่แตะข้อมูลจริง)
node experiments/e2e/mock-supabase.mjs

# เทอร์มินัล 2 — build แอปให้ชี้ mock แล้วเปิดเซิร์ฟเวอร์
VITE_SUPABASE_URL=http://localhost:54321 VITE_SUPABASE_PUBLISHABLE_KEY=mock npm run build
npx vite preview --host 127.0.0.1 --port 8080

# เทอร์มินัล 3 — รันการทดสอบ
node experiments/e2e/drive.mjs      # E2E ฟีเจอร์หลัก 16 ขั้น
node experiments/e2e/drive2.mjs     # E2E ภาค 2: หนี้สิน/เป้าหมายออม/CSV/API ล่ม/จอแคบ 13 ขั้น
node experiments/e2e/xlsx-e2e.mjs   # E2E นำเข้า Excel ครบวงจร 11 ขั้น
node experiments/e2e/perf-test.mjs  # วัดความเร็วโหลด 5 รอบ
```

> ⚠️ `drive.mjs` มีเช็ค "วิดเจ็ตซ่อนเมื่อไม่มีข้อมูล" — ต้องรันบน mock ที่เพิ่งเปิดใหม่ (DB ว่าง)
> ถ้ารันสลับชุด ให้รีสตาร์ต `mock-supabase.mjs` ก่อนแต่ละชุด

> ⚠️ สำคัญ: อย่าลืม build ใหม่แบบปกติ (`npm run build` เฉย ๆ) ก่อน deploy จริง
> เพราะ build ข้างบนชี้ฐานข้อมูลจำลอง

> สคริปต์คาดหวัง Chromium ที่ `/opt/pw-browsers/chromium` — ถ้าอยู่ที่อื่น
> แก้ `executablePath` ในไฟล์ .mjs หรือใช้ `npx playwright install chromium` แล้วชี้ path ตามเครื่อง

## ทำไมใช้ฐานข้อมูลจำลอง (เขียนในเล่มได้)

- **ตัดตัวแปรแทรกซ้อน**: ความเร็ว/ความพร้อมของอินเทอร์เน็ตและเซิร์ฟเวอร์จริงไม่มากวนผล
- **ทำซ้ำได้ 100%**: ทุกครั้งเริ่มจากฐานข้อมูลว่างเหมือนกัน
- **ปลอดภัย**: ไม่มีทางกระทบข้อมูลผู้ใช้จริง
- ข้อจำกัดที่ต้องเขียนไว้: ตัวเลขความเร็วเป็นเครือข่ายภายในเครื่อง ของจริงบนมือถือ+4G จะช้ากว่า (ใช้เปรียบเทียบเชิงสัมพัทธ์/ก่อน-หลังปรับปรุงได้ แต่ไม่ใช่ตัวเลขสัมบูรณ์ของผู้ใช้จริง)
