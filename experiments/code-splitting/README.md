# การทดลองที่ 9: ผลของเทคนิคการแบ่งโหลดโค้ด (Code Splitting) ต่อความเร็วในการโหลดเว็บแอปพลิเคชัน

การทดลองเชิงจัดกระทำ (true experiment) ที่**ไม่ต้องใช้คนทดลองเลย** — สร้างแอปเดียวกัน 2 เวอร์ชัน
ที่ต่างกันแค่ตัวแปรเดียว (มี/ไม่มีเทคนิค code splitting) แล้ววัดความเร็วแข่งกันบนเครือข่ายจำลอง
ตอบคำถามว่า "ที่แอปเราโหลดเร็ว เร็วเพราะเทคนิคที่ใช้จริงหรือไม่ และเร็วขึ้นกี่เท่า"

## กรอบวิธีการทางวิทยาศาสตร์

| หัวข้อ | เนื้อหา |
|---|---|
| **สมมติฐาน** | เวอร์ชันที่ใช้เทคนิค code splitting แสดงผลหน้าแรกเร็วกว่าและโหลดข้อมูลน้อยกว่าเวอร์ชันที่ไม่ใช้ ในทุกชนิดเครือข่าย |
| **ตัวแปรต้น** | การใช้เทคนิค code splitting (2 ระดับ: A = ใช้ / B = ไม่ใช้) |
| **ตัวแปรตาม** | (1) เวลาแสดงผลแรก FCP (ms) (2) ปริมาณข้อมูลที่โหลดหน้าแรก (KB) |
| **ตัวแปรควบคุม** | ซอร์สโค้ดเดียวกันทุกบรรทัด (ต่างเฉพาะกลไกโหลด) · เครื่อง/เบราว์เซอร์เดียวกัน · เครือข่ายจำลองโปรไฟล์เดียวกัน (3G/4G/WiFi ตามค่ามาตรฐาน Chrome) · วัด 5 รอบ/เงื่อนไข ไม่มี cache ทุกรอบ · เซิร์ฟเวอร์ชนิดเดียวกัน |

**เทคนิคที่ "ปิด" ในเวอร์ชัน B** (3 ชั้น — แปลงอัตโนมัติด้วยสคริปต์ ไม่แก้ตรรกะใด ๆ):
1. React.lazy ของ 32 หน้าจอ → เปลี่ยนเป็น import ปกติ (ทุกหน้ารวมในไฟล์เดียว)
2. dynamic import ของไลบรารีหนัก (อ่าน Excel 429KB, สร้าง PDF 415KB) → import ปกติ
3. การแยก vendor chunks ใน Vite (react-vendor/radix/supabase) → ปิด

ผลเชิงโครงสร้าง: เวอร์ชัน A แบ่งเป็น 123 ไฟล์ (โหลดเฉพาะที่ใช้) ส่วนเวอร์ชัน B เหลือไฟล์หลัก
ก้อนเดียวขนาด 2.6 MB ที่ต้องโหลดทั้งหมดก่อนเริ่มแสดงผล

## ผลการทดลอง (รันจริง 12 ก.ค. 2569 — ค่าเฉลี่ย 5 รอบ/เงื่อนไข)

| เครือข่าย | A มี code splitting | B ไม่มี | A เร็วกว่า |
|---|---|---|---|
| 3G | **1,442 ms** | 4,543 ms | 3.2 เท่า |
| 4G | **399 ms** | 1,145 ms | 2.9 เท่า |
| WiFi | **151 ms** | 568 ms | 3.8 เท่า |
| ข้อมูลที่โหลดหน้าแรก | **~209 KB** | ~800 KB | ลดลง 74% |

**สรุป**: ยอมรับสมมติฐาน — เทคนิค code splitting ทำให้แอปแสดงผลเร็วขึ้นประมาณ 3 เท่า
และลดปริมาณข้อมูลที่ต้องโหลดลง 74% ในทุกชนิดเครือข่าย ผลต่างยิ่งมีความหมายบนเครือข่ายช้า
(3G: จาก 4.5 วินาที เหลือ 1.4 วินาที) ซึ่งเป็นบริบทการใช้งานจริงของนักเรียนที่ใช้เน็ตมือถือ

## วิธีทำซ้ำ

```sh
# 1) build เวอร์ชัน A (โค้ดปัจจุบัน)
VITE_SUPABASE_URL=http://localhost:54321 VITE_SUPABASE_PUBLISHABLE_KEY=mock npx vite build --outDir dist-a
# 2) แปลงโค้ดปิด code splitting (ดูสคริปต์แปลงใน git history ของการทดลองนี้) แล้ว build --outDir dist-b
# 3) เปิด mock + เสิร์ฟทั้งคู่ (พอร์ต 8080/8081) แล้ววัด:
node experiments/e2e/mock-supabase.mjs &
npx vite preview --outDir dist-a --host 127.0.0.1 --port 8080 &
npx vite preview --outDir dist-b --host 127.0.0.1 --port 8081 &
node experiments/code-splitting/measure.mjs
```

ข้อมูลดิบรายรอบ: `codesplit-results.csv`
