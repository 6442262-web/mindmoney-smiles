# การทดลองที่ 1: ความแม่นยำของ AI แยกรายการจากข้อความแชท

> 📁 โครงงานนี้มี **3 การทดลอง** เอาไว้วัดแอปคนละมุม — ดูภาพรวมท้ายไฟล์ `app-usability/README.md`
> 1. **ความแม่นของ AI** (โฟลเดอร์นี้) — วัด "สมอง" ของระบบ
> 2. **ประสิทธิภาพเมื่อผู้ใช้ใช้จริง** → `experiments/app-usability/` — วัดว่าคนใช้แล้วดีขึ้นจริงไหม (เวลา/error/SUS + สถิติ)
> 3. **ความน่าเชื่อถือของระบบ** → `npx vitest run` — % เคสทดสอบอัตโนมัติที่ผ่าน

การทดลองสำหรับรูปเล่มโครงงาน — วัดว่า AI ดึง **การตัดสินใจบันทึก / ประเภทรับ-จ่าย / จำนวนเงิน / หมวดหมู่** ถูกต้องกี่ % จากประโยคภาษาไทย 100 ประโยค แล้วเปรียบเทียบ **ก่อน/หลังปรับปรุง prompt**

## ไฟล์ในโฟลเดอร์นี้

| ไฟล์ | คืออะไร |
|---|---|
| `chat-test-set.json` | ชุดทดสอบ 100 ประโยค 9 กลุ่ม (เลขอารบิก, ตัวหนังสือไทย, เลขไทย, ตัวย่อ k, รายรับ, สแลง, คำถาม, ไม่มีจำนวน, หลายรายการ) พร้อมเฉลย |
| `run-chat-accuracy.mjs` | สคริปต์ยิงทุกประโยคไปที่ AI จริง แล้วให้คะแนนอัตโนมัติ + ออก CSV |
| `baseline-chat-transaction.ts` | โค้ดฟังก์ชันแชท **เวอร์ชันก่อนปรับ prompt** (ใช้ทำกลุ่มควบคุม) |

## ขั้นตอนการทดลอง (ก่อน → หลัง)

ต้องมี: เครื่องที่ clone repo นี้ + `npm install` แล้ว + บัญชีทดสอบในแอป

### รอบที่ 1 — "ก่อนปรับปรุง" (baseline)
1. Deploy ฟังก์ชันเวอร์ชันเก่า: copy โค้ดจาก `baseline-chat-transaction.ts` ไปวางทับใน
   Supabase Dashboard → Edge Functions → chat-transaction → Editor → Deploy
2. รัน:
   ```sh
   TEST_EMAIL=อีเมลของคุณ TEST_PASSWORD=รหัสผ่าน node experiments/run-chat-accuracy.mjs
   ```
3. เก็บไฟล์ `results-*.csv` ที่ได้ → เปลี่ยนชื่อเป็น `before.csv`

### รอบที่ 2 — "หลังปรับปรุง"
1. Deploy ฟังก์ชันเวอร์ชันใหม่: copy จาก `supabase/functions/chat-transaction/index.ts` ไปวางแล้ว Deploy
2. รันคำสั่งเดิมอีกครั้ง → เก็บผลเป็น `after.csv`

### รายงานผล
สคริปต์พิมพ์ตารางสรุป % ความถูกต้องรวม + แยกตามกลุ่มให้เลย เอาไปทำกราฟแท่งเทียบก่อน/หลังใน Excel/Google Sheets ได้ทันที (CSV เปิดใน Excel ได้ ภาษาไทยไม่เพี้ยน)

## หมายเหตุสำคัญ

- **ใช้เวลา ~9 นาที/รอบ** (หน่วง 5 วิ/ประโยคกัน rate limit ของ Gemini free tier) — ถ้าโดน rate limit ให้เพิ่ม `DELAY_MS=8000`
- ลองน้อย ๆ ก่อนได้: `LIMIT=10 TEST_EMAIL=... TEST_PASSWORD=... node experiments/run-chat-accuracy.mjs`
- สคริปต์**ไม่บันทึกรายการลง database** (แค่ถาม AI) — ใช้บัญชีจริงได้ปลอดภัย
- ระเบียบวิธีที่ควรเขียนในเล่ม: รันชุดเดียวกันทั้งสองรอบ, ตัวแปรควบคุม = โมเดล/ชุดทดสอบ/บัญชีเดียวกัน, ตัวแปรต้น = prompt, ตัวแปรตาม = % ความถูกต้อง 4 ด้าน

## สิ่งที่ปรับปรุงใน prompt เวอร์ชันใหม่ (ใส่ในเล่มได้)

1. กฎแปลงจำนวนเงินทุกรูปแบบ: เลขไทย ๕๐, ตัวหนังสือ "สามร้อย/พันนึง/หมื่นห้า", ตัวย่อ "1.5k", สตางค์
2. กฎแยกรับ-จ่ายละเอียดขึ้น (แม่ให้เงิน/ถูกหวย/ได้คืน = รายรับ ฯลฯ)
3. ตารางจับคู่คำ → หมวดหมู่ (เกม→บันเทิง, วิน→ค่าเดินทาง, 7-11 ของกิน→อาหาร)
4. กฎกันพลาด: คำถามที่มีตัวเลขต้องไม่บันทึก, หลายรายการให้ขอทีละรายการ, ไม่มีจำนวนให้ถามกลับ
5. ตัวอย่างสอน (few-shot) 12 ตัวอย่างครอบคลุมเคสยาก
