# การทดลองที่ 6: เปรียบเทียบ 3 วิธีอ่านสลิปโอนเงิน (QR vs AI vs ไฮบริด)

การทดลองไฮไลต์ของโครงงาน — เพราะทดสอบ **การออกแบบเชิงวิศวกรรมของเราเอง**:
แอป MoneyMind ไม่ได้ใช้ AI อ่านสลิปอย่างเดียว แต่ถอด **QR มาตรฐานตรวจสอบสลิปของธนาคารไทย**
ในเครื่องควบคู่ไปด้วย (ฟรี, เป๊ะ 100% เมื่อเจอ, ได้เลขอ้างอิงไว้ยืนยันสลิปแท้)
การทดลองนี้พิสูจน์ว่าการผสมสองวิธี (ไฮบริด) ดีกว่าการใช้วิธีใดวิธีหนึ่งจริงไหม — ด้วยตัวเลข

## กรอบวิธีการทางวิทยาศาสตร์

| หัวข้อ | เนื้อหา |
|---|---|
| **สมมติฐาน** | วิธีไฮบริดให้ "บันทึกที่สมบูรณ์และยืนยันได้" มากกว่า QR อย่างเดียวหรือ AI อย่างเดียว |
| **ตัวแปรต้น** | วิธีอ่านสลิป (A: QR / B: AI / C: ไฮบริด) |
| **ตัวแปรตาม** | % ความถูกต้องของ ธนาคาร · ยอดเงิน · วันที่, % "บันทึกสมบูรณ์" (ถูกครบ 3 อย่าง), % ยืนยันเลขอ้างอิงได้, เวลาต่อสลิป |
| **ตัวแปรควบคุม** | สลิปชุดเดียวกันทุกวิธี · รูปเดียวกันไฟล์เดียวกัน · รันในคราวเดียวกัน (สคริปต์ทำ 3 วิธีต่อรูปพร้อมกัน) |
| **กลุ่มตัวอย่าง** | สลิปจริง **≥ 30 ใบ** จากหลายธนาคาร (อย่างน้อย 3–4 ธนาคาร) และหลายสภาพ (ชัด/เบลอ/สกรีนช็อต/ถ่ายจากจอ) |

จุดแข็งเชิงวิธีวิทยา (เขียนในเล่ม): ทั้ง 3 วิธีอ่าน **ไฟล์รูปเดียวกันเป๊ะ** ในรอบเดียวกัน → ไม่มีตัวแปรแทรกซ้อนจากรูปหรือเวลา และการให้คะแนนเป็นอัตโนมัติจากเฉลยที่กรอกไว้ล่วงหน้า → ไม่มีอคติผู้ตรวจ

## ⚠️ จริยธรรม/ความเป็นส่วนตัว (ต้องเขียนในเล่มด้วย)

- ใช้เฉพาะสลิปของตัวเอง/ครอบครัว/เพื่อนที่**ยินยอม**
- ชื่อ-เลขบัญชีบนสลิปเป็นข้อมูลส่วนบุคคล — ในรูปเล่มให้เบลอก่อนแปะ
- **ห้าม commit รูปสลิปขึ้น git** (โฟลเดอร์ `slips/` ถูก .gitignore ไว้แล้ว)

## ขั้นตอน

1. **Deploy ฟังก์ชัน `scan-slip` เวอร์ชันล่าสุดก่อน** (Supabase Dashboard → Edge Functions → Editor)
   — เวอร์ชันล่าสุดเพิ่มให้ AI ตอบ `sendingBank` ซึ่งการทดลองนี้ต้องใช้
2. รวบรวมสลิป วางใน `experiments/slip-scan/slips/` ตั้งชื่อ `slip-001.jpg`, `slip-002.png`, …
3. คัดลอก `answers-template.csv` → `answers.csv` แล้วกรอกเฉลยจริงของแต่ละใบ:
   - `bank_code` = รหัสธนาคารต้นทาง: 002 กรุงเทพ · 004 กสิกรไทย · 006 กรุงไทย · 011 ttb · 014 ไทยพาณิชย์ · 022 CIMB · 024 UOB · 025 กรุงศรี · 030 ออมสิน · 033 ธอส. · 034 ธ.ก.ส. · 067 ทิสโก้ · 069 KKP · 073 LH Bank
   - `date` = วันที่โอนจริง รูปแบบ ค.ศ. YYYY-MM-DD · `amount` = ยอดจริง
4. ทดสอบระบบก่อน (ไม่แตะ AI): `node experiments/slip-scan/run-slip-experiment.mjs --selftest`
5. รันจริง:
   ```sh
   TEST_EMAIL=อีเมล TEST_PASSWORD=รหัสผ่าน node experiments/slip-scan/run-slip-experiment.mjs
   ```
   ลองน้อยรูปก่อนได้: เติม `LIMIT=3` หน้าคำสั่ง · โดน rate limit ให้เพิ่ม `DELAY_MS=8000`

สคริปต์จะพิมพ์ตารางสรุป % ทุกตัวชี้วัด × 3 วิธี และเขียน `slip-results-วันที่.csv` รายใบสำหรับทำกราฟ

## ผลที่คาดว่าจะเห็น (กรอบการอภิปราย)

- **QR อย่างเดียว**: ธนาคาร/เลขอ้างอิงเป๊ะเมื่อเจอ QR แต่ *อ่านยอดเงินไม่ได้เลย* (มาตรฐาน QR สลิปไทยไม่บรรจุยอดเงิน) และจะไม่เจอ QR ในรูปที่ครอปตัด/เบลอมาก → "บันทึกสมบูรณ์" = 0%
- **AI อย่างเดียว**: อ่านได้ทุกช่องแต่มีโอกาสผิด (ยอด/วันที่/ธนาคาร) และยืนยันสลิปแท้ไม่ได้
- **ไฮบริด**: ธนาคารควรถูก ≥ AI (เพราะใช้ QR ทับเมื่อเจอ) + ยืนยันเลขอ้างอิงได้เท่าอัตราเจอ QR
  → ประเด็นอภิปราย: จุดแข็งของแต่ละวิธีเสริมกันตรงจุดอ่อนของอีกวิธีพอดี
